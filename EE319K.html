about the project.


This project was my first hands-on experience with building a video game from scratch. Within my Intro to Embedded Systems class, students paired up and were instructed to make a video game with a given set of requirements (use of collision detection, screen display, etc.). My lab partner and I coded a game called "Abduct the Cow," whereupon a single player had to shoot a light beam at a cow. When the player captured the cow, the player was introduced to a new and more rigorous level.







my role.


Within this project, I was in charge of collision and bound detection as well as level control. I developed code to detect if the lightbeam launched by the player hit the cow and if the moving cow had reached the LCD screen border (whereupon the cow would reverse its direction of movement). In addition, I developed the level control system by creating different instances of the game with memory for remaining lives and increasing difficulty.







the challenge.


Creating a reliable level control system was the most difficult part of this project. With many changing variables - speed of cow across the screen, remaining lives, number of obstacles - a large struggle of creating the game was finding which of these factors was causing problems. By isolating each of these factors and reducing the size of the problem, my team and I were able to detect the source of our issues.
